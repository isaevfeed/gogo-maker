# Gogo Maker

CLI утилита для быстрого создания Go-проектов из шаблонов.

## Возможности

- Создание проектов из Git-репозиториев-шаблонов
- Автоматическая замена имени модуля во всех файлах проекта
- Поддержка различных типов проектов (HTTP-сервер, gRPC-сервер и др.)
- Кроссплатформенность (macOS, Linux)

## Установка

```bash
make build
```

или

```bash
go build -o gogo-maker ./cmd/gogo-maker
```

## Использование

### Инициализация

При первом запуске необходимо инициализировать конфигурационный файл:

```bash
gogo-maker init
```

Конфиг будет создан в директории `~/.gogo-maker/config.yaml`. Отредактируйте его, чтобы добавить свои шаблоны проектов.

### Создание проекта

```bash
gogo-maker create <тип> <имя> [директория]
```

**Параметры:**
- `тип` — тип проекта из конфига (например, `http-server`, `grpc-server`)
- `имя` — имя нового проекта (будет использовано как имя модуля)
- `директория` — опционально, путь для создания проекта (по умолчанию — текущая директория)

**Примеры:**

```bash
# Создать HTTP-сервер с именем my-api в текущей директории
gogo-maker create http-server my-api

# Создать проект в указанной директории
gogo-maker create http-server my-api ./projects
```

## Конфигурация

Конфигурационный файл `~/.gogo-maker/config.yaml`:

```yaml
reps:
  - name: Best HTTP server
    url: https://github.com/user/golang-http-server-template.git
    type: http-server
  - name: Best gRPC server
    url: https://github.com/user/golang-rpc-server-template.git
    type: grpc-server
```

**Поля репозитория:**
- `name` — описание шаблона (для отображения)
- `url` — URL Git-репозитория с шаблоном
- `type` — идентификатор типа проекта (используется в команде `create`)

## Как работает замена модуля

При создании проекта утилита автоматически:

1. Клонирует репозиторий-шаблон во временную директорию
2. Удаляет директорию `.git`
3. Переименовывает `cmd/<template-name>` в `cmd/<project-name>`
4. Заменяет имя модуля в файлах:
   - `.go` файлы (импорты)
   - `go.mod`, `go.sum`
   - `Dockerfile`, `Dockerfile.*`
   - `Makefile`
   - `docker-compose.yml`, `docker-compose.yaml`
   - `.dockerignore`, `.gitignore`
   - `README.md`
   - Файлы `.yaml`, `.yml`, `.json`, `.toml`, `.sh`, `.md`, `.txt`
5. Копирует проект в указанную директорию

## Требования к шаблону

Репозиторий-шаблон должен:
- Иметь имя модуля в `go.mod`, совпадающее с именем репозитория (без `.git`)
- Содержать исполняемый код в `cmd/<template-name>/`
- Корректно использовать импорты внутреннего модуля
